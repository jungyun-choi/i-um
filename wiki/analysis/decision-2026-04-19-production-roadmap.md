## 배경

MVP 완성 (AI 일기 생성, 타임라인, 편집/삭제) 이후 프로덕션 수준으로 끌어올리기 위한 방향을 논의함.
핵심 원칙: **"AI가 써주는 일기"** 차별점을 극한까지 강화하는 방향으로 집중.

---

## 냉정한 현실 체크 — 지금 당장 해결해야 할 문제

| 문제 | 영향 |
|------|------|
| 푸시 알림 없음 | 일기 완성돼도 부모가 모름 → 앱 열 이유 없어짐 |
| 온보딩 없음 | 첫 실행 시 뭘 해야 할지 모름 |
| 생성 실패 시 재시도 UX 없음 | 실패하면 그냥 멈춤, 사용자 이탈 |
| 데이터 백업 불안감 | 소중한 사진/일기가 날아갈 수 있다는 인식 |

## 아이디어 전체 목록

### 탐색 개선
- **월별 달력 뷰**: 달력에서 날짜 탭 → 그날 일기로 이동
- **연령별 퀵점프**: "0살", "1살" 탭
- **검색**: 장소명·마일스톤 텍스트 검색

### 감동 만들기
- **AI 월간 레터**: 매달 말 그달 하이라이트를 편지 형식으로 자동 생성 ⭐ 킬러
- **"오늘로부터 1년 전" 메모리 카드**: 홈 화면에 1년 전 기억 노출
- **마일스톤 알림**: "생후 100일이에요!" 자동 알림
- **AI 연간 레터**: 연말 "올해의 [아이이름]" 종합 편지

### 공유 & 가족 참여
- **배우자 초대**: 읽기 전용 → 댓글 → 공동 작성 단계적 확장
- **조부모 공유 링크**: 앱 설치 없이 웹으로 열람
- **가족 댓글**: 가족 구성원이 일기에 반응 남기기

### 분석 & 재미 요소 (데이터 충분히 쌓인 후)
- **가족 성향 분석**: 방문 장소 패턴으로 "바다파 가족" 등 캐릭터 생성
- **아이 성장 그래프**: 마일스톤 달성 타임라인 시각화

### 수익화
- **프리미엄 구독**: 월 일기 제한 → 무제한 (가격: 월 4,900원 예상)
- **포토북 출력**: 연간·마일스톤 포토북 인쇄 주문 (파트너십 필요)
- **하이라이트 영상**: 월말 자동 영상 생성 (프리미엄) — 원래 이음 핵심 비전

## 채택 로드맵

### Phase 0 — 기본기 완성 (지금 ~ 2주)
- [ ] 푸시 알림 (일기 완성 시 Expo Push Notification)
- [ ] 월별 달력/퀵점프 탐색
- [ ] 생성 실패 시 재시도 UX

### Phase 1 — 감동 만들기 (1~2달)
- [ ] AI 월간 레터 (매달 1일 자동 발송)
- [ ] "오늘로부터 1년 전" 메모리 카드
- [ ] 마일스톤 알림 자동화

### Phase 2 — 공유 & 확장 (2~3달)
- [ ] 배우자 초대 (읽기 전용)
- [ ] 조부모 공유 링크 (웹 뷰)
- [ ] AI 연간 레터 (연말)

### Phase 3 — 수익화 (3~6달)
- [ ] 프리미엄 구독 도입
- [ ] 포토북 출력 파트너십
- [ ] 하이라이트 영상 (프리미엄)
