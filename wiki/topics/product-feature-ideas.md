## 탐색 개선

### 월별 달력 뷰
- **문제**: 일기가 수십 개 쌓이면 스크롤로 특정 날짜 찾기 불가능
- **해결**: 상단에 월 달력 컴포넌트 → 일기 있는 날 점 표시 → 탭하면 해당 일기로 스크롤
- **구현 복잡도**: 낮음 (타임라인 데이터를 날짜 맵으로 변환, 달력 라이브러리 사용)
- **우선순위**: 🔴 최우선 (데이터 쌓이면 없으면 못 씀)

### 연령별 퀵점프
- **구현**: 타임라인 상단에 "0살 / 1살 / 2살" 탭 → 해당 시기 첫 일기로 점프
- **구현 복잡도**: 낮음
- **우선순위**: 🟡 중간

### 검색
- **구현**: 장소명(location_name), 마일스톤 텍스트, 일기 내용 전문 검색
- **구현 복잡도**: 중간 (Supabase full-text search or ilike)
- **우선순위**: 🟡 중간

## AI 감동 기능

### AI 월간 레터 ⭐ 킬러 기능
- **컨셉**: 매달 말일 자동으로 그달 일기들을 요약한 편지 생성 + 푸시 발송
  - "4월의 민준이 — 이번 달 민준이는 처음으로 혼자 앉았어요. 총 12번의 이야기가 기록됐어요."
- **구현**: Bull Queue cron job → 달 일기 목록 조회 → Claude로 종합 레터 생성 → 알림 발송 → 앱 내 "레터함"에 저장
- **구현 복잡도**: 중간
- **차별성**: FamilyAlbum·Google Photos 둘 다 없는 기능. 이음의 핵심 가치 확장
- **우선순위**: 🔴 최우선 (Phase 1 첫 번째)

### "오늘로부터 1년 전" 메모리 카드
- **컨셉**: 홈 화면 상단에 "1년 전 오늘" 카드 노출 (Google Photos Memories 유사)
- **구현**: 타임라인 쿼리 시 exactly 1년 전 ±3일 범위 조회 → 있으면 카드 렌더
- **구현 복잡도**: 낮음
- **우선순위**: 🟢 낮음 (데이터가 1년 쌓여야 의미 있음)

### 마일스톤 알림 자동화
- **컨셉**: 생후 100일, 200일, 1주년 등 자동 푸시 + 축하 일기 제안
- **구현**: 아이 birth_date 기준 cron → 특정 일수 도달 시 푸시
- **구현 복잡도**: 낮음
- **우선순위**: 🟡 중간

### AI 연간 레터
- **컨셉**: 매년 12월 31일 그해 전체 일기를 종합한 "올해의 [아이이름]" 편지
- **구현**: 월간 레터와 유사, 범위만 1년으로 확장
- **우선순위**: 🟢 낮음 (데이터 1년 후)

## 가족 공유 & 참여

### 배우자 초대
- **컨셉**: 이메일로 초대 → 같은 아이 일기 공유
- **단계적 확장**: 읽기 전용 → 댓글 → 공동 일기 작성
- **구현**: children 테이블에 co_parent_id 추가 or 별도 family_members 테이블
- **구현 복잡도**: 중간~높음 (권한 관리, 충돌 처리)
- **우선순위**: 🟡 중간 (Phase 2)

### 조부모 공유 링크
- **컨셉**: 앱 설치 없이 웹 브라우저로 열람 가능한 읽기 전용 링크
- **구현**: 토큰 기반 public 뷰어 페이지 (Next.js or 모바일 웹뷰)
- **구현 복잡도**: 높음 (웹 뷰어 별도 구현 필요)
- **우선순위**: 🟡 중간 (조부모 사용성 핵심)

## 수익화

### 프리미엄 구독
- **모델**: 무료 (월 5개 일기) / 프리미엄 무제한 (월 4,900원 예상)
- **구현**: Supabase RLS + RevenueCat (iOS/Android 인앱 결제)
- **우선순위**: 🟡 중간 (Phase 3 초반)

### 포토북 출력
- **모델**: AI가 레이아웃 자동 구성 → 파트너사에 인쇄 주문
- **주의**: 인쇄 파트너 계약, 배송, CS 오퍼레이션이 생김 → 팀 규모 필요
- **우선순위**: 🟢 낮음 (시리즈A 이후)

### 하이라이트 영상 자동 생성
- **컨셉**: 월말 사진들로 BGM 포함 15초~1분 영상 자동 생성
- **주의**: AI 영상 API 비용 큼, 품질 편차 심함 (Runway/Sora)
- **비고**: 이음 원래 핵심 비전 — 비용/품질 해결되면 최우선
- **우선순위**: 🟢 낮음 (Phase 3)
