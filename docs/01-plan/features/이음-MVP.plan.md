# 이음(i-um) MVP — Plan Document

## Executive Summary

| 항목 | 내용 |
|------|------|
| Feature | 이음 MVP — AI 제로에포트 육아 기록 앱 |
| 시작일 | 2026-04-19 |
| 목표 출시 | 2026년 하반기 (베타) |
| 개발 기간 | 약 3~4개월 |

### Value Delivered (4-Perspective)

| 관점 | 내용 |
|------|------|
| **Problem** | 부모는 아이의 모든 순간을 기록하고 싶지만, 매번 직접 일기 쓸 시간이 없다. 수천 장의 사진이 쌓여도 아무 이야기가 되지 않는다. |
| **Solution** | 사진만 찍어서 저장하면 AI가 육아 일기를 대신 써주고, 아이의 성장을 일대기로 엮어준다. 부모가 할 일은 사진 찍기뿐. |
| **Function & UX Effect** | 사진첩 자동 분석 → AI 일기 생성 → 타임라인/일대기 뷰. 매일 밤 "오늘 일기가 완성됐어요" 알림. 감동적인 서술로 부모 감정 연결. |
| **Core Value** | 시장 최초 "AI가 쓰는 육아 일기" — 어떤 경쟁사도 하지 않는 텍스트 서사 자동 생성으로 독보적 포지션 확보. |

## 1. 배경 및 목표

### 1.1 문제 정의
- 부모의 99%가 아이 사진을 찍지만, 대부분 기록으로 남기지 못함
- 기존 앱(FamilyAlbum, Tinybeans, 쑥쑥찰칵)은 모두 **글쓰기를 부모에게 맡김**
- 시장조사 결과: "사진 → 단락형 일기 자동 생성" 기능을 가진 앱이 현재 전무

### 1.2 목표
1. 사진 한 장에서 감성적인 육아 일기 서술을 자동 생성
2. 아이의 성장을 챕터 구조의 일대기로 자동 엮기
3. 부모의 노력을 0에 가깝게 줄이기 (Zero-effort)

### 1.3 성공 기준 (MVP)
- AI 일기 생성 품질: 부모 만족도 70% 이상 ("이 글이 내 아이 이야기 같다")
- 사진 업로드 → 일기 생성까지 30초 이내
- 베타 사용자 D7 리텐션 40% 이상

## 2. MVP 기능 범위

### 2.1 MVP In-scope (반드시 포함)

#### [F1] 사진 임포트 & 자동 분석
- 카메라롤 접근 권한 요청 및 사진 선택
- 배치 업로드 (최대 50장/회)
- AI 분석: 얼굴(아이/부모/조부모), 장소(집/야외/식당 등), 표정, 계절
- EXIF 메타데이터 추출 (날짜, GPS)
- 아이 나이 자동 계산 (출생일 기준)

#### [F2] AI 일기 생성 (핵심 차별점)
- 사진 분석 결과 + 메타데이터 → Claude API (멀티모달)
- 출력: 2~4문단 감성 육아 일기 (한국어)
- 서술 스타일: 부모 시점 ("오늘 민준이는..."), 따뜻하고 감성적인 톤
- 한국 육아 마일스톤 반영 (백일, 돌, 첫걸음 등 자동 인식)
- 생성된 일기는 편집 가능

#### [F3] 타임라인 뷰
- 날짜순 사진 + 일기 피드
- 월별/챕터별 구분 (출생~100일, 100일~돌 등)
- 카드형 UI: 사진 썸네일 + 일기 첫 문장

#### [F4] 아이 프로필
- 이름, 생년월일, 성별 입력
- 아이 얼굴 등록 (이후 자동 분류 기준)
- 복수 아이 지원 (최대 3명)

#### [F5] 마일스톤 자동 인식
- 백일(D+100), 돌(D+365) 자동 감지 및 특별 표시
- AI가 "첫걸음", "첫말" 등을 사진/일기 맥락에서 감지하면 마일스톤 뱃지
- 마일스톤 목록 뷰 (아이의 주요 순간 모음)

#### [F6] 기본 인증
- 이메일/비밀번호 가입 및 로그인
- (선택) 카카오 소셜 로그인

### 2.2 MVP Out-of-scope (v2 이후)

| 기능 | 이유 |
|------|------|
| AI 하이라이트 영상 생성 | 복잡도 높음, v2에서 추가 |
| 가족 공유 (조부모 등) | 인증·권한 시스템 필요, v2 |
| 온디바이스 AI | 초기엔 클라우드 API로 빠른 개발 |
| Android 앱 | iOS 먼저 검증 후 확장 |
| 포토북/인쇄 연동 | v3 수익화 단계 |
| 푸시 알림 고도화 | 기본 알림만 MVP 포함 |

## 3. 기술 스택

### 3.1 Frontend
- **React Native (Expo)** — iOS 우선, 추후 Android 확장
- Expo Router (파일 기반 라우팅)
- React Query (서버 상태 관리)
- Zustand (클라이언트 상태)
- NativeWind (스타일링)

### 3.2 Backend
- **Node.js + Express** (API 서버)
- PostgreSQL (사용자·아이 프로필, 일기 데이터)
- AWS S3 (사진 원본 저장)
- Redis (AI 생성 작업 큐)

### 3.3 AI
- **Claude API (claude-opus-4-7)** — 멀티모달 (사진 분석 + 일기 생성)
- 프롬프트 캐싱으로 API 비용 최적화
- 배치 처리: 여러 사진을 한 번의 API 호출로 처리

### 3.4 인프라 (MVP)
- Vercel (백엔드 API, 간단한 배포)
- AWS S3 (사진 저장)
- Supabase (PostgreSQL + Auth, 초기 빠른 셋업)

### 3.5 기술 선택 근거

| 결정 | 이유 |
|------|------|
| React Native (Expo) | iOS/Android 코드 공유, Expo의 빠른 개발 속도 |
| Claude API | 멀티모달 + 한국어 감성 서술 품질 최고 |
| Supabase | Auth + DB를 빠르게 셋업, MVP에 최적 |
| iOS 우선 | 한국 육아 앱 주 사용자 iPhone 비중 높음 |

## 4. 개발 로드맵

### Phase 1 — 핵심 루프 (6주)
> 목표: "사진 찍기 → 일기 생성" 핵심 루프 동작

- [ ] 프로젝트 초기화 (React Native + Supabase + Node.js)
- [ ] 아이 프로필 생성 화면
- [ ] 카메라롤 접근 + 사진 선택 UI
- [ ] 사진 S3 업로드 파이프라인
- [ ] Claude API 멀티모달 일기 생성 (백엔드)
- [ ] 일기 결과 화면 (사진 + 생성된 텍스트)
- [ ] 기본 인증 (이메일/비밀번호)

### Phase 2 — 타임라인 & 마일스톤 (4주)
> 목표: 일기들이 "이야기"로 보이는 뷰

- [ ] 타임라인 피드 UI (카드형)
- [ ] 월별/챕터 구분
- [ ] 마일스톤 자동 인식 (백일·돌)
- [ ] 일기 편집 기능
- [ ] 마일스톤 뷰 페이지

### Phase 3 — 폴리시 & 베타 준비 (3주)
> 목표: 실제 부모가 쓸 수 있는 품질

- [ ] 프롬프트 튜닝 (일기 품질 개선)
- [ ] 로딩/에러 UX 개선
- [ ] 기본 푸시 알림 ("오늘 일기가 완성됐어요")
- [ ] TestFlight 베타 배포
- [ ] 지인 5~10가족 베타 테스트

## 5. AI 일기 생성 설계

### 5.1 프롬프트 구조

```
[시스템 컨텍스트]
- 아이 이름: {name}
- 생년월일: {birthdate} → 현재 나이: {age_days}일 ({age_text})
- 특별 날짜 여부: {milestone if applicable}

[사진 분석 요청]
첨부된 사진들을 분석하고, 한국 부모가 아이를 위해 쓰는 따뜻한 육아 일기를 작성해주세요.
- 사진 날짜: {date}
- 장소 정보: {gps_location if available}

[출력 형식]
- 2~4문단, 150~300자
- 부모 시점, 현재형
- 한국 육아 감성 (따뜻하고 구체적)
- 마일스톤이면 특별히 언급
```

### 5.2 비용 추정

| 항목 | 수치 |
|------|------|
| 사진 1장당 토큰 | ~1,000 토큰 (이미지 포함) |
| 일기 생성 출력 | ~300 토큰 |
| Claude API 비용 | ~$0.01/일기 생성 (opus-4-7 기준) |
| 월 100명 사용자, 1일 3장 | 약 $90/월 |

## 6. 리스크 및 대응

| 리스크 | 가능성 | 대응 |
|--------|--------|------|
| AI 일기 품질이 어색함 | 높음 | 프롬프트 반복 튜닝, 부모 편집 기능 |
